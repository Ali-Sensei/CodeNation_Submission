Completing Digital Projects Successfully

Technology has improved to a point where it can be overwhelming to incorporate it into business. Judging from my past experience and from what I have learned so far, I would like to mention the technologies below that have seen a measure of success.

Planning and Preparation

Before doing anything, it is important to have a plan of action, and to be able to share this with teams as well as getting feedback on what the teams have produced so far in regards to completing tasks and getting research done.

Trello

Trello is an online share space that acts like a whiteboard with post it notes, except that this is online version that is freely editable to those that have been given permission to a board, whether that be public or private.

Information can be viewed at a glance at a high-level view, then cards can be delved into by adding comments, deadlines, checklists with clickable items, allowing a team to fully collaborate on a project from beginning the end. I even used Trello to plan the making of this website! By cutting down large tasks into smaller and more manageable tasks, the workload can be more easily divided and handled, making for a fun and rewarding project.

The Building Phase

The phase where it all comes together. Here, people use the software that they need in order to accomplish something, whether that is using Microsoft PowerPoint to present slides, using Unity to create the latest and greatest computer and video games, or using a text editor to create the applications of tomorrow. There are far too many applications to list here, but a few are worth mentioning at least:

Online Storage

Carrying out digital projects often involves a number of digital files which are stored locally on a computer, which is fine if one person is exclusively involved with that project. But there are situations where files need be shared with each other, which can be done by sending emails back and forth between members, but this is convoluted and disorganized and a large folder full of emails of different documents can quickly become too much to handle.

This is why a shared space such as Google Drive or Microsoft’s OneDrive can be so useful. It is like having an online drive where people that have permission can access the drive and update and modify the files. Of course, this is dependent on the permissions that have been set up by an administrator allowing for secure access to files. Files on a digital drive can be updated and viewed in real-time, and even allows for multiple people to work on a document simultaneously! Files on these shared spaces can include version control, allowing previous changes to be viewed, and in the more dire circumstances, can be rolled back to. Because the files are stored online, you are able to access the files from any computer and you can take comfort in the fact that files are not lost if a virus or other circumstance affects a computer because the files are not even stored locally, but stored online accessible from any computer...with the right permissions of course!

GitHub

GitHub is very similar to the above section of ‘Online Drive’ because of the fact that information is stored in the cloud (i.e. online). However, GitHub is primarily used for the storage of code of a variety of programming languages with the inclusion of version control. Because of the more narrowed focus on code, it is useful in a variety of ways.

Firstly, publishing code to GitHub allows others to view your code and comment on it, which is highly valuable for getting information on unforeseen issues such as conflicts between code and the ability of the code to work independently of other code. GitHub is one of the largest repositories around, thus publishing code allows for a greater deal of public exposure. The inclusion of version control allows the tracking of changes, which is incredibly important when multiple people are working on a project. The cloud aspect of GitHub allows people in different locations to work on a project seamlessly and with reduced conflict.

If the project is open-source, then GitHub is available for free and it allows members of a team or individuals to set up wikis and issue trackers. Members can fork a project, make changes, and then submit ‘pull requests’ so that the changes can be incorporated into the overall project.

Whatever Gets The Job Done

In the end, a successful project is one that has been planned thoroughly, shared appropriately with a team, and through collaboration, a final digital project is completed successfully.
